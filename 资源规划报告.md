## 二、资源规划目标

本资源规划报告用于明确农产品价格采集监控平台实施过程中所需的软件、工具、数据、人力、硬件和时间资源，帮助小组合理分工、控制进度并降低项目实施风险。

根据实训要求，平台需要完成数据采集、数据清洗、数据存储、统计分析、机器学习预测、可视化展示、测试验收和文档提交等任务，因此资源规划重点包括：

1. 明确本组当前可落地实现所需的软件与开发环境。
2. 明确完整实训平台扩展所需的 Hadoop、Spark、Kafka、Redis 等组件资源。
3. 明确农产品价格数据、气象数据和辅助数据来源。
4. 明确小组成员在各阶段的任务分工。
5. 估算课程实操成本和完整平台扩展成本。
6. 制定进度安排和风险保障措施。

## 十一、风险与资源保障措施

| 风险 | 可能影响 | 保障措施 |
| --- | --- | --- |
| 数据获取不足 | 影响分析和模型训练 | 准备多个公开数据源，必要时使用课程样例数据补充 |
| 爬虫页面变化 | 采集脚本失效 | 保留 CSV 数据备份，并将爬虫作为补充采集方式 |
| 大数据组件部署困难 | 影响实训进度 | 当前使用本地可运行方案，HDFS/Kafka/Spark 作为扩展说明 |
| MySQL 环境异常 | 入库和查询受影响 | 提供 SQLite 降级方案和自动化测试 |
| 成员任务进度不一致 | 影响最终整合 | 每周同步进度，负责人及时调整分工 |
| 模型效果不理想 | 预测误差较大 | 增加时间、地区、气象特征，并设置基线模型对比 |
| 文件版本混乱 | 造成报告或代码丢失 | 使用统一目录和 Git/网盘备份，重要文件保留历史版本 |

## 十二、资源规划结论

本项目所需资源以开源软件、公开数据和小组成员时间投入为主，当前本地实操直接资金成本较低，适合在课程周期内完成。技术资源方面，Python、pandas、MySQL、SQLite、Flask、Vue、ECharts、scikit-learn 等工具能够支撑本组完成可运行的农产品价格采集监控平台原型。
