## 五、入库流程说明

### 5.1 实训平台标准流程

农产品价格采集监控平台完整数据链路如下：

```text
爬虫/API/CSV 数据采集
        ↓
Kafka 消息队列或本地原始文件
        ↓
数据清洗：去重、缺失值处理、价格单位转换、时间格式统一
        ↓
HDFS/CSV 保存原始数据和清洗后数据
        ↓
MySQL 保存结构化价格数据、市场数据、预警数据和预测结果
        ↓
后端接口和前端可视化页面查询展示
```

### 5.2 本组当前实操流程

本组的数据入库由 DBImporter 模块完成，主要步骤如下：

1. 执行 backend/schema.sql 创建 price_data 和 weather_data 表。
2. 读取 data/processed/processed_price.csv，调用 import_price_data 导入 price_data 表。
3. 读取 data/processed/processed_weather.csv，调用 import_weather_data 导入 weather_data 表。
4. 每批数据使用 pandas to_sql 进行批量写入，批大小为 1000 行。
5. 调用 verify_import(table_name, expected_count) 检查数据库表记录数是否与 CSV 数据行数一致。
6. 如果 MySQL 连接失败，则自动降级到 SQLite，并使用兼容表结构继续导入。

## 十、常用查询语句记录

### 10.1 查询价格表总记录数

```sql
SELECT COUNT(*) AS price_count
FROM price_data;
```

预期结果：2169 条。

### 10.2 查询气象表总记录数

```sql
SELECT COUNT(*) AS weather_count
FROM weather_data;
```

预期结果：723 条。

### 10.3 按产品查询平均价格

```sql
SELECT
    product_name,
    COUNT(*) AS record_count,
    ROUND(AVG(average_price), 2) AS avg_price
FROM price_data
GROUP BY product_name
ORDER BY product_name;
```

### 10.4 按地区查询价格数据量

```sql
SELECT
    region,
    COUNT(*) AS record_count
FROM price_data
GROUP BY region
ORDER BY region;
```

### 10.5 价格与气象关联查询

```sql
SELECT
    p.product_name,
    p.region,
    p.date,
    p.average_price,
    w.average_temperature,
    w.rainfall,
    w.humidity
FROM price_data p
JOIN weather_data w
  ON p.region = w.region
 AND p.date = w.date;
```

预期结果：2169 条。

### 10.6 价格波动查询扩展示例

```sql
SELECT
    product_name,
    region,
    date,
    average_price,
    average_price - LAG(average_price) OVER (
        PARTITION BY product_name, region
        ORDER BY date
    ) AS price_change
FROM price_data;
```

该查询可用于后续识别价格波动异常情况，对应实训平台中的价格监控和预警功能。

## 十一、异常场景与处理记录

| 异常场景 | 处理方式 | 测试结论 |
| --- | --- | --- |
| MySQL 服务不可用 | 使用 SQLite 降级数据库继续建表和导入 | 不影响本地测试和演示流程 |
| CSV 文件缺少必要字段 | import_price_data 或 import_weather_data 抛出 ValueError | 可避免错误数据写入数据库 |
| 输入 DataFrame 为空 | 导入方法返回 0 并跳过写入 | 可避免空数据造成异常中断 |
| CSV 存在多余字段 | DBImporter 仅保留表结构中定义的字段 | 不影响数据库表结构稳定性 |
| 入库后记录数不一致 | verify_import 返回 False | 可作为入库失败的判定依据 |
| 价格单位不统一 | 清洗阶段统一为元/公斤 | 保证统计分析口径一致 |
| 日期格式不统一 | 清洗阶段统一为 yyyy-MM-dd | 保证按日期查询和关联正确 |
